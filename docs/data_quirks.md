# Data Quirks: NPDES / ICIS (ECHO)

Interesting quirks/potential errors across all NPDES data sets. 

| File                                        | Variable                                                    | Issue                                                                                                                                                                                                                                                                                                | Panel impact                                                                                                                     | Severity                      |
| ------------------------------------------- | ----------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------- | ----------------------------- |
| `NPDES_FORMAL_ENFORCEMENT_ACTIONS`          | `FED_PENALTY_ASSESSED_AMT`, `SETTLEMENT_ENTERED_DATE`       | One enforcement action (single `ACTIVITY_ID`) is stored as one row per permit it touches: same fine, same date repeated. PRASA = 135 rows of `$1,024,427` *(measured)*; Deepwater Horizon $3.35B recorded as lead + non-lead records with different state/compliance allocations (verified vs ECHO). | Summing penalties/actions naively double-counts multi-permit cases; aggregate/dedupe at `ACTIVITY_ID`.                           | High                          |
| `ICIS_FACILITIES`                           | `FACILITY_UIN` ↔ `NPDES_ID`                                 | 10.5% of FRS IDs map to >1 NPDES ID *(measured)*, inflated by generic umbrellas (one ND UIN = 361 unrelated permits).                                                                                                                                                                                | Dangerous universe-wide, but near 1:1 for major-individual — only 84 facilities hold >1 permit (one has 4, four have 3, rest 2). | High (universe) / Low (panel) |
| `NPDES_CATCHMENTS`                          | catchment record / `NHDPLUSID`                              | Covers inland surface waters only: coastal/offshore facilities have no record and silently drop from inner-join spatial merges (no error).                                                                                                                                                           | ~7% (549) of major-individual permits (coastal/marine) vanish from catchment joins → anti-join to catch them.                    | Medium                        |
| `ICIS_PERMITS`                              | `PERMIT_STATUS_CODE`, `EXPIRATION_DATE`, `TERMINATION_DATE` | No facility open/close field; existence inferred from permit lifecycle. `ADC` (admin-continued, ~243k) = "expired" but still active, so `EXPIRATION_DATE` overstates closures. Permit ≠ facility.                                                                                                    | Entry/exit derived from permit dates; must treat ADC/EXP-continued as active or you mis-date exits.                              | Medium                        |
| `NPDES_NAICS`, `NPDES_SICS`                 | `NAICS_CODE`, `SIC_CODE`                                    | NAICS ~65% missing; SIC ~99% for majors but ~39% of *all* permits have neither *(measured)*. Missingness flat across vintages (not date-gated).                                                                                                                                                      | NAICS unusable panel-wide → use SIC (~99% in panel). Newest entrants worst-coded.                                                | Medium                        |
| `NPDES_NAICS` + `NPDES_SICS`                | `NAICS_CODE` ↔ `SIC_CODE`                                   | No clean crosswalk: many-to-many (NAICS 221320 → 88 SICs; SIC 1542 → 115 NAICS) *(measured)*.                                                                                                                                                                                                        | Can't convert/reconcile the two systems; pick one and use its primary code.                                                      | Medium                        |
| bulk enforcement files (vs case-report API) | SEP value, compliance-action cost, cost recovery            | Absent from bulk files: only in the case-report API (PRASA: $1M penalty vs $195M compliance cost).                                                                                                                                                                                                   | Bulk penalty totals understate true case weight.                                                                                 | Low                           |
| `NPDES_FORMAL/INFORMAL_ENFORCEMENT_ACTIONS` | `FED_PENALTY_ASSESSED_AMT`, `STATE_LOCAL_PENALTY_AMT`       | Mostly missing/zero: the majority of actions are informal (no penalty).                                                                                                                                                                                                                              | Don't use penalty `$` as a general enforcement outcome; use action counts / SNC flags.                                           | Medium                        |
| `NPDES_DMRS`                                | `DMR_VALUE_NMBR` (non-receipt)                              | Coverage majors ~84% vs minors ~7% (FY2025) *(measured)*; the 2016 eRule pulls minors in → non-stationary.                                                                                                                                                                                           | Majors well-covered; if adding DMR/violations, control for/flag the ~2016 reporting break.                                       | Medium (if using DMR)         |
| `NPDES_EFF_VIOLATIONS`                      | whole file                                                  | Derived & lossy — only violating rows, no compliant reports. Absent facility-year = "no violation," NOT "no discharge."                                                                                                                                                                              | Can't compute rates/averages; violation counts only.                                                                             | Medium                        |
| `NPDES_EFF_VIOLATIONS`                      | `EXCEEDENCE_PCT`, `RNC_*`, `NODI_CODE`, `DAYS_LATE`         | Populated only for their specific violation subtype (conditionally defined, not truly missing).                                                                                                                                                                                                      | Condition on `VIOLATION_TYPE_CODE` before aggregating or means/sums mislead.                                                     | Medium                        |
| all CSVs (esp. `ICIS_FACILITIES`, `ICIS_PERMITS`) | quoted text fields (`FACILITY_NAME`, `RAD_WBD_HUC12S`…) | Legal-but-tricky CSV — embedded commas (`"BP EXPLORATION&PRODUCTION, INC"`) and embedded newlines inside quoted fields (a multi-line `RAD_WBD_HUC12S` value spilled "Waterloo Municipal Airport" fragments) — break naive `cut -d,`/`awk -F,` and line-by-line splitting, silently shifting columns or spawning fake "rows." The data is fine; proper parsers reassemble it. | Use a real CSV parser (`read.csv`/`fread`/Python `csv`); never delimiter- or line-split. | Medium (technical) |
| `ICIS_PERMITS`                              | whole file                                                  | Embedded NUL bytes → R `read.csv` warns / Python `csv` errors.                                                                                                                                                                                                                                       | `skipNul=TRUE` (R) or strip `\x00` (Python).                                                                                     | Medium (technical)            |
| `ICIS_PERMITS`                              | `EXTERNAL_PERMIT_NMBR`, `VERSION_NMBR`                      | Individual permits keep the same NPDES_ID across reissuance (versioned, ~52% >1 version); general coverages get renumbered each cycle (`NDR100000`→`NDR110000`).                                                                                                                                     | Individual IDs are stable → coherent time series (general renumber not in panel).                                                | Low                           |
| `ICIS_PERMITS`                              | `MAJOR_MINOR_STATUS_FLAG`                                   | ~3.6% of rows blank *(measured)*.                                                                                                                                                                                                                                                                    | Those facilities can't be classified; drop from major-restricted sample.                                                         | Low                           |
| `NPDES_NAICS`, `NPDES_SICS`                 | `NAICS_CODE`, `SIC_CODE`                                    | Snapshots (not time-varying); also ~49k facilities have >1 SIC, ~9k >1 NAICS *(measured)* (multi-activity sites).                                                                                                                                                                                    | Treat industry as time-invariant; collapse with the primary code.                                                                | Low                           |
| `ICIS_FACILITIES`                           | `GEOCODE_LATITUDE`/`LONGITUDE`                              | General/minor sites often un-geocoded → city/county-centroid or placeholder coords; many "facilities" at one point. Conflicting geocodes for offshore/corporate permits (BP: office vs offshore ~370 km apart; inland-TX "99999" placeholder).                                                       | Only ~0.5% of major-individual have invalid geocodes *(measured)* — good within panel, bad outside. Check `LATLONG_TYPE`.        | Low (panel) / High (general)  |
| `NPDES_PERM_FEATURE_COORDS`                 | `PERM_FEATURE_NMBR` / coords                                | A permit can be a spatial cloud — `GMG290110` = 76 outfalls over ~510 km *(measured)*. Individual permits: 44% multi-outfall but clustered at one site.                                                                                                                                              | Individual outfalls cluster → collapsing to a point is fine; extreme spread is general/offshore.                                 | Low (panel)                   |
| multiple files                              | `MAJOR_MINOR_FLAG` vs `MAJOR_MINOR_STATUS_FLAG`             | Column names differ across files (outfalls layer vs permits); ECHO renames columns.                                                                                                                                                                                                                  | Join defensively on candidate names.                                                                                             | Low                           |
| `NPDES_EFF_VIOLATIONS`                      | `ADJUSTED_DMR_VALUE_NMBR`                                   | Essentially always empty — 6 of 46,361,587 rows *(measured)*; effluent trading is negligible.                                                                                                                                                                                                        | Ignore the field.                                                                                                                | Low                           |
| `NPDES_ATTAINS_AU_SUMMARIES`                | `REPORTINGCYCLE` / impairment                               | Snapshot, not a time series — only the latest cycle per unit (0 of 813k unit-pairs have >1) *(measured)*; no impairment history.                                                                                                                                                                     | Water quality = time-invariant covariate; can't study change over the window.                                                    | Low                           |
| `NPDES_CATCHMENTS`                          | (no time column)                                            | Static facility→catchment spatial crosswalk (physical hydrography).                                                                                                                                                                                                                                  | Time-invariant; fine to join once.                                                                                               | Low                           |
| `ICIS_PERMITS`                              | `EFFECTIVE_DATE`, `ISSUE_DATE`, `ORIGINAL_ISSUE_DATE`       | 1,987 of 99,940 individual (`NPD`) permits (~2%) have all three start-date fields blank in *every* version — genuinely blank, not an `mdy()` parse failure (0 unparseable) *(measured)*. No opening date → can't be placed on a timeline. Exceeds the ~0.8% date-missingness quoted for permits overall. | Dropped from the facility-by-month panel for lacking an opening month; all 1,987 are minor (**0 ever-major**), so no effect on the major-individual panel. | Low (panel)                   |
| `NPDES_INFORMAL_ENFORCEMENT_ACTIONS`        | whole file / `ENF_IDENTIFIER`                               | **345,822 of 821,977 rows (42%) are byte-identical duplicates** — all 11 fields equal (same `ENF_IDENTIFIER`, `NPDES_ID`, `ENF_TYPE_CODE`, `ACHIEVED_DATE`, `OFFICIAL_FLG`) *(measured)*. NOT legitimate multi-permit/type fan-out: after de-duping, only 812 actions have any genuine multi-row structure. The formal file has **0** exact dups. | Panel counts informal **per raw row** by design (script `05_add_enforcement.R`, Assumption 1a) so the dups count: `N_INFORMAL_ACTIONS` = 93,470 rows vs 56,356 distinct actions (~1.7×). Formal stays distinct-action → informal & formal counts are **not** on the same footing (don't sum/compare directly). Any row-level reuse of the raw file must `unique()` first (drops 345,822). | Medium |
| `NPDES_INFORMAL_ENFORCEMENT_ACTIONS` (panel-derived) | `N_INFORMAL_ACTIONS` vs `N_OFFICIAL_INFORMAL` + `N_UNOFFICIAL_INFORMAL` | 45 facility-months contain an informal action carrying **both** an official and an unofficial flag row (all **EPA Region 4** AL/FL/KY/MS/NC/SC/TN, **2005–2009**, always a single LOVWL) *(measured)*. Under the **old distinct-action** count these components exceeded the total by 1; since informal is now counted **per raw row** (script `05`, Assumption 1a), each flag row counts in both the total and its component, so `N_OFFICIAL_INFORMAL + N_UNOFFICIAL_INFORMAL == N_INFORMAL_ACTIONS` **holds in every row** (verified in the 05 run log). Formal still partitions cleanly too. | No longer a panel mismatch under per-row counting; the raw double-flagging persists but is harmless. | Low (panel) |
| `NPDES_DMRS` / `NPDES_LIMITS` (TSS, param `00530`) | `LIMIT_VALUE_QUALIFIER_CODE` | For TSS the qualifier is a near-constant `<=` (ceiling): **691,109 of 691,889 numeric-limit rows (99.9%)**, with only 707 `=` and 73 `<` (0.1% combined); the 62,144 blanks are monitor-only rows (no numeric limit) *(measured, FY2025 EXO eff-gross monthly-avg slice)*. Both concentration and mass limits are `<=`. | Uniform `<=` → carries no variation for TSS, so **not a panel column**; instead fold the ceiling direction into the violation test (exceedance = `reported > limit`) and QC the ~780 non-`<=` rows (treat `<` as `<=`; decide explicitly on the 707 `=`). Uniformity is also what makes averaging/summing limits across outfalls coherent (can't mix a ceiling with a floor). **Holds for TSS only** — parameters with minimum limits (pH, DO, % removal) use `>=` (violation = `reported < limit`); `LIMIT_VALUE_QUALIFIER_CODE` **must** be carried explicitly if the parameter set expands. | Low (TSS) / High (multi-param) |
